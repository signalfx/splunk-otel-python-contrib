Adding AI Defense to Your App

A guide for integrating Cisco AI Defense — from zero to working

1. Your App Today (No AI Defense)

This is a typical GenAI app. It takes user input, sends it to an LLM, and returns a response. No security checks.

2. Two Ways to Add AI Defense

3. Gateway Mode — Step by Step

EASY  Minimal changes. Best starting point.

1

Add one line of instrumentation

# Add this BEFORE importing your LLM framework from opentelemetry.instrumentation.aidefense import AIDefenseInstrumentor AIDefenseInstrumentor().instrument() # Then import your framework as usual from langchain_openai import ChatOpenAI

This auto-captures the X-Cisco-AI-Defense-Event-Id header from gateway responses and adds it to your existing spans.

2

Change the base_url to the AI Defense Gateway

# Before (direct to OpenAI) llm = ChatOpenAI( model="gpt-4o-mini", base_url="https://api.openai.com/v1", api_key="sk-...", ) # After (through AI Defense Gateway) llm = ChatOpenAI( model="gpt-4o-mini", base_url=os.environ["AI_DEFENSE_GATEWAY_URL"], api_key="sk-...", # same key, gateway forwards it )
3

Set the environment variable

export AI_DEFENSE_GATEWAY_URL="https://us.gateway.aidefense.security.cisco.com/{tenant}/connections/{conn}/v1"

Get this URL from the AI Defense management portal. That's it — you're done.

That's the entire integration for Gateway mode. Two lines of code + one env var. Your app now has AI security, and every LLM span in Splunk O11y will have a gen_ai.security.event_id when a violation is detected.

4. API Mode — Step by Step

MORE CONTROL  More code changes, but you control the blocking behavior.

1

Install the AI Defense SDK

pip install cisco-aidefense-sdk
2

Add instrumentation (same as Gateway)

from opentelemetry.instrumentation.aidefense import AIDefenseInstrumentor AIDefenseInstrumentor().instrument() # auto-wraps inspect_prompt()
3

Create the AI Defense client

from aidefense.runtime import ChatInspectionClient client = ChatInspectionClient(api_key=os.environ["AI_DEFENSE_API_KEY"])
4

Add security checks before LLM calls

# Before (no security) def handle_user_input(user_input): response = llm.invoke(user_input) return response # After (with AI Defense) def handle_user_input(user_input): # Check with AI Defense first result = client.inspect_prompt(user_input) if not result.is_safe: # You decide what to do: # - Block completely # - Return a warning # - Log and continue anyway return "Sorry, that request was flagged for security reasons." # Only reaches LLM if AI Defense says it's safe response = llm.invoke(user_input) return response

You add this check wherever you want security. Could be once at the top, or before every agent, or only for user-facing inputs.

5

Set the environment variable

export AI_DEFENSE_API_KEY="your-api-key-from-ai-defense-portal"

5. Which Mode Should I Pick?

Recommendation for the AI Assistant team: Start with Gateway Mode. Get the URL from the AI Defense portal, change one line, add the instrumentor, and you'll have security + observability in 5 minutes. Switch to API Mode later if you need to block prompts before they reach the LLM.
