How the Demo App Works

A plain-English walkthrough of the Multi-Agent Travel Planner

1. What Is This App?

It's a trip planning app that uses 3 AI agents working together. You give it a destination, and it finds flights, hotels, and activities for you.

Each agent is a specialist — it has a specific job, a specific tool it can use, and an LLM (GPT-4o-mini) brain to reason with. The agents don't talk to each other directly — they share a common state (like a shared notepad).

2. What Is LangGraph?

LangGraph is a Python library that lets you build AI workflows as a flowchart (graph). Instead of writing if/else chains, you define nodes and edges, and LangGraph executes them in order.

In code, building the graph looks like this:

graph = StateGraph(PlannerState) — create a graph with shared state
graph.add_node("flight_specialist", flight_specialist_node) — register a function as a node
graph.add_conditional_edges(START, should_continue) — add decision logic between nodes
compiled_app = graph.compile() — compile it into a runnable app
compiled_app.stream(initial_state) — run it step by step

6. Where Does the CronJob Come In?

The app itself is just a Python script — run it and it exits. The CronJob is a Kubernetes resource that we created separately to run it on a schedule.

7. What Are the "Mock Tools"?

The agents have tools — functions they can call to look up information. In a real app, these would call booking APIs. In our demo, they return fake data.

Why mock? The demo exists to show AI Defense + OpenTelemetry integration, not to actually book trips. Using mock tools means the demo is self-contained — no API keys for booking services, no cost, deterministic results every time.
