3. Inside an Agent Node (Step by Step)

1

Security Check — check_security()

Calls security_client.inspect_prompt(request) via the Cisco AI Defense SDK. The AIDefenseInstrumentor auto-wraps this and creates a chat cisco-ai-defense span with gen_ai.security.event_id.

result = security_client.inspect_prompt(request) if not result.is_safe: # Return immediately — agent never invoked return False, result.event_id
If blocked: State is updated with blocked_by_security=True and security_event_id. The function returns early — no LLM call, no invoke_agent span.
2

Create LLM — create_llm()

Gets an OAuth2 token from Cisco's identity service, constructs ChatOpenAI pointing to chat-ai.cisco.com.

token = token_manager.get_token() # OAuth2: id.cisco.com → JWT return ChatOpenAI( model="gpt-4o-mini", base_url=OAuth2TokenManager.get_llm_base_url(model), # → https://chat-ai.cisco.com/openai/deployments/gpt-4o-mini api_key="placeholder", default_headers={"api-key": token}, # JWT in header )
3

Create Agent — create_agent()

Creates a LangChain agent with tools and config. The .with_config() sets metadata that becomes span attributes.

agent = create_agent( model=llm, tools=[mock_search_flights], system_prompt="You are a flight specialist...", name="flight_specialist", ).with_config({ "run_name": "flight_specialist", # → span name "tags": ["agent"], # → span tags "metadata": {"agent_name": "..."}, # → gen_ai.agent.name })
4

Invoke Agent — agent.invoke()

Runs the LangGraph agent. The LangchainInstrumentor auto-creates spans for every step.

result = agent.invoke({"messages": [HumanMessage(content=request)]}) # Auto-generated spans by LangchainInstrumentor: # invoke_agent flight_specialist ← gen_ai.agent.name ✓ # step model # chat gpt-4o-mini ← gen_ai.usage.input_tokens ✓ # step tools # execute_tool mock_search_... ← gen_ai.tool.name ✓ # step model # chat gpt-4o-mini ← final answer
